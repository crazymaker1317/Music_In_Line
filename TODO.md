## 1. 음표 배열 알고리즘 고도화

**필요성**: 프로토타입의 기초 로직은 선의 모양이 음악에 반영되는 것을 보여주지만, 음악적으로 자연스러운 결과를 보장하지 않습니다.

**프로토타입 한계**:
- 기울기 변화에 따른 음표 종류 결정이 단순함
- 마디 내 리듬 패턴이 기계적으로 느껴질 수 있음
- 쉼표(rest) 생성 로직이 없음

**작업 내용**:
- [ ] 기울기 변화에 따른 음표 종류 결정 로직 세분화
- [ ] 마디 내 리듬 패턴의 자연스러움 향상 (약박/강박 고려)
- [ ] 쉼표(rest) 생성 로직 추가
- [ ] 다이나믹(강약) 표현 추가 (기울기 크기 → 음량)

## 2. 캔버스 드로잉 UX 개선

**필요성**: 현재 캔버스는 기본적인 그리기만 지원하며, 초등학생 사용자를 위한 가이드가 부족합니다.

**프로토타입 한계**:
- 좌→우 방향 제약의 실시간 피드백이 없음 (데이터 처리 단계에서 보정)
- 마디 구분선이나 음높이 가이드가 캔버스에 표시되지 않음

**작업 내용**:
- [ ] 좌→우 방향 제약의 실시간 시각적 피드백
- [ ] 드로잉 가이드라인 표시 (마디 구분선, 음높이 가이드)
- [ ] 터치/태블릿 입력 최적화
- [ ] 선 지우기/다시 그리기 기능 강화

---

## 3. 악보 시각화 완성

**필요성**: 텍스트 기반 음표 정보보다 시각적 악보가 교육적으로 훨씬 효과적입니다.

**프로토타입 한계**:
- `music21` 라이브러리 연동이 미구현
- 텍스트 기반으로만 음표 정보를 표시

**작업 내용**:
- [ ] `music21` 또는 다른 라이브러리를 활용한 정식 악보 렌더링
- [ ] 실시간 악보 업데이트
- [ ] 악보에 마디 번호, 박자표 등 표시
